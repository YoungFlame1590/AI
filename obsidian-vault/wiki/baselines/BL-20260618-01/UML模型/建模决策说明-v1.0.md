# 建模决策说明

## 1. 涉众到 Actor 映射
| 原始涉众 | UML Actor | 映射依据 |
|---|---|---|
| 客户 | 客户 | 订单发起方、文件上传方、支付与取件主体 |
| 门店店员 | 门店店员 | 一线接单录入、初步质检、异常一键标记执行者 |
| 门店店长 | 门店店长 | 门店权限兜底、折扣阈值内审批、生产看板监控 |
| 总部运营管理员 | 总部运营管理员 | 跨店调度终审、价格模板管控、异常闭环监督 |
| 财务人员 | 财务人员 | 日报表核对、多渠道流水勾稽、退款与发票审计 |
| 配送/外协人员 | 配送/外协人员 | 跨店调货、外协工艺执行、任务清单签收 |
| 系统管理员 | 系统管理员 | 权限拦截配置、审批流节点维护、日志与审计策略设定 |

## 2. 用例合并与拆分理由
| 用例名称 | 处理策略 | 业务理由 |
|---|---|---|
| 接单与录入信息 + 自动报价与折扣审批 | 拆分但用 <<include>> 关联 | 响应Q02/Q04冲突。录入操作需独立考核≤3分钟时效，报价与折扣审批涉及权限拦截与总部审批流，逻辑独立但强依赖，故拆分后通过包含关系串联。 |
| 订单质检与状态流转 + 异常处理与人工兜底 | 拆分并用 <<extend>> 关联 | 响应Q01/Q07冲突。质检是转“生产中”的前置必要条件（正常路径），异常兜底仅在质检失败或设备异常时触发（扩展路径），避免状态机逻辑互斥。 |
| 生产派单与工位推送 + 配送任务接收与执行 | 拆分并用 <<include>> 关联 | 响应Q05不一致。自动派单依赖负荷校验（≤60%），跨店/外协场景必须包含配送任务生成与清单同步，确保首屏固定显示三项关键数据。 |
| 日结对账与报表生成 + 异常处理与人工兜底 | 拆分并用 <<include>> 关联 | 响应Q03矛盾。财务对账强依赖兜底操作产生的“调整前后金额”与“责任人”字段，将对账逻辑独立为用例，明确数据源与审计闭环。 |

## 3. 活动图路径覆盖清单
| 活动图名称 | 正常路径 (Normal) | 异常路径 (Abnormal) | 边界路径 (Boundary) |
|---|---|---|---|
| 接单报价 | 信息完整→自动报价→支付成功→转待质检 | 信息缺失→联系客户补充；支付失败→保持待付款 | 折扣超店长权限→触发总部线上审批流 |
| 质检与派单 | 质检通过→负荷达标(≤60%)→生成工单直推工位 | 质检不通过→退回待修改/待协商→通知客户 | 负荷超标或工艺不匹配→触发跨店调度/外协派单 |
| 异常兜底 | 常规异常→自动捕获元数据→按SOP流转 | 需改派/拆单→执行干预；拦截改价→仅允许分摊或审批 | 总部终审补录业务原因→生成路径对比表→≤4小时闭环 |
| 日结对账 | 正常订单汇总→账实一致→确认入账归档 | 存在改价/拆单→提取金额与责任人；退款差异→匹配原单 | 定位差异订单至操作员→人工核查修正→重新触发对账 |

## 4. 核心冲突建模化解策略
- **Q01 状态流转冲突**：活动图明确“初步质检”为转“生产中”的强前置节点。系统自动解析与负荷校验仅在质检通过后执行，消除逻辑互斥。
- **Q02/Q03 定价与改价冲突**：引入权限阈值校验与总部审批流。系统拦截直接改价，仅允许“金额分摊”或“特殊审批”，同时强制记录调整前后金额满足财务对账。
- **Q05 派发模式不一致**：采用“自动解析+负荷校验”为主路径，失败或负荷超标时降级为“跨店/外协派单”，配送首屏固定展示地址/时限/明细，实现半自动闭环。
- **Q07 极简操作与强审计冲突**：通过系统自动捕获操作人、IP、精确到秒时间戳，店员仅需勾选“异常原因分类标签”（≤2步），兼顾效率与合规审计。
