## 1. 引言

### 1.1 目的
本文档旨在为“图文快印门店连锁管理系统”提供完整、可追溯、可验证的软件需求规格说明。文档作为开发、测试、验收及后期维护的唯一基准，确保各涉众对系统功能、性能、数据及接口的理解一致。

### 1.2 范围
本系统覆盖订单接单、自动报价、生产质检、负荷派单、跨店/外协调度、多渠道收款对账、权限管控与操作审计等核心业务。适用涉众包括：客户、门店店员、门店店长、总部运营管理员、财务人员、配送/外协人员、系统管理员。系统不包含财务ERP核心账务核算、硬件设备底层驱动开发及第三方物流干线调度。

### 1.3 定义与缩略语
| 术语/缩写 | 定义 |
|---|---|
| SLA | 服务级别协议，此处指订单从提交到完成生产的承诺时效 |
| QC | 质检（Quality Check），指前台对文件可打开性、规格合规性的初步校验 |
| 负荷≤60% | 设备空闲产能指标，用于自动派单的第一优先级阈值 |
| 2步操作 | 异常标记交互上限，含点击标记与选择分类标签 |
| 日结 | 每日闭店自动触发的跨渠道流水勾稽与报表生成动作 |

## 2. 总体描述

### 2.1 产品视角
本系统为B/S架构连锁管理平台，前端支持Web管理端与移动端H5/小程序，后端对接支付网关（微信/支付宝/企业网银）、短信通知服务、门店工位屏推送接口及外协工厂数据同步通道。系统作为门店日常运营与总部集中管控的中枢，向下接管门店本地工单流转，向上聚合财务对账与审计日志。

### 2.2 产品功能概述
系统提供订单全生命周期管理：客户提交文件与订单后，店员≤3分钟完成信息录入；系统自动带出标准报价，支持店长阈值内折扣与总部审批流；支付成功后进入“待质检”状态，前台QC通过后系统解析工艺并校验设备负荷（≤60%），达标则直推工位屏，超标则触发跨店/外协派单；异常兜底支持≤2步极简标记与自动元数据捕获，4小时内闭环；每日闭店自动生成对账报表，差异定位至订单与操作员；全量操作强制留痕，权限按角色与门店严格隔离。

### 2.3 用户特征
| 涉众角色 | 技术熟练度 | 核心操作目标 |
|---|---|---|
| 客户 | 低/中 | 上传文件、查看报价、在线支付、接收进度提醒、一键复购 |
| 门店店员 | 中 | ≤3分钟完成接单录入、执行前台QC、≤2步标记异常 |
| 门店店长 | 中 | 监控积压/逾期看板、审批阈值内折扣、执行2步兜底改派 |
| 总部运营管理员 | 高 | 制定全市价格模板、终审超权限折扣、监督≤4小时异常闭环 |
| 财务人员 | 高 | 核对“门店+渠道”流水、处理发票与退款、30分钟内完成日结 |
| 配送/外协人员 | 低/中 | 查看首屏固定3项数据、执行调货/外协、在线签收 |
| 系统管理员 | 高 | 配置权限边界、维护审批节点、监控日志与异常告警 |

### 2.4 约束条件
- 价格管控：严禁随意修改基础单价，分店仅可微调本地附加费（每日限3次）。
- 时效约束：普通订单30分钟内流转，急件≤1小时；审批流生效延迟≤2分钟；异常闭环≤4小时；日结对账≤30分钟。
- 操作约束：一线录入≤3分钟；异常标记≤2步；人工兜底最多允许2步。
- 合规约束：所有干预强制记录操作人、精确到秒时间戳及IP；全量/未脱敏导出需总部审批。

### 2.5 假设与依赖
- 假设门店网络带宽稳定，支持大文件上传与实时状态推送。
- 依赖第三方支付网关API返回准确交易流水与手续费数据。
- 依赖门店设备IoT或手工上报接口提供实时负荷与耗材库存数据。
- 假设《可承接规格清单》已作为系统校验白名单录入基础数据库。

### 3.2 非功能需求
| 类别 | 需求描述 | 验收标准 |
|---|---|---|
| 性能 | 系统并发处理能力与响应延迟 | 单店日均处理≥200单时，页面加载与接口响应≤2秒；审批流状态同步延迟≤2分钟；高峰期权限锁死阻断接单时，支持应急降级模式 |
| 可用性 | 系统运行稳定性 | 核心业务模块（接单、支付、派单）可用性≥99.9%；月度异常派单/改派15-20起时，系统自动重试与人工兜底机制保障业务不中断 |
| 安全性 | 权限隔离与数据审计 | 实行最小权限原则；所有配置变更与兜底操作100%可审计；敏感数据（客户联系方式、支付流水）存储加密，传输采用TLS 1.2+ |
| 易用性 | 交互效率与防错设计 | 一线接单录入≤3分钟；异常标记≤2步；易错字段（纸张克重、精确页数、后道工艺）提供默认值与二次确认弹窗；工位屏推送无多余层级 |
| 可靠性 | 数据一致性与备份 | 支付流水与系统订单100%匹配；日报表差异可穿透至订单与操作员；每日凌晨自动全量备份，恢复时间目标(RTO)≤4小时 |

### 3.3 接口需求
| 接口名称 | 交互对象 | 协议/格式 | 核心数据项 | 频率/触发条件 |
|---|---|---|---|---|
| 支付网关接口 | 微信/支付宝/企业网银 | HTTPS/JSON | 订单号、应收金额、渠道单号、手续费、支付状态 | 客户提交支付时实时调用，闭店对账时拉取流水 |
| 工位屏推送接口 | 门店生产设备/终端屏 | WebSocket/MQTT | 工单号、工艺要求、SLA截止时间、优先级 | 质检通过且负荷达标后即时推送 |
| 短信/通知服务 | 运营商网关 | HTTPS/REST | 客户手机号、订单状态、取件提醒（提前15分钟） | 状态变更时异步触发，取件前定时触发 |
| 外协厂数据同步 | 合作加工厂系统 | API/CSV | 物品明细、工艺要求、最晚完工时间、签收状态 | 跨店/外协任务生成时同步，完工后回调更新 |
| 设备负荷上报接口 | 门店IoT网关/手工录入端 | HTTPS/JSON | 设备ID、当前负荷百分比、核心耗材可用量 | 每5分钟轮询或人工手动刷新 |
