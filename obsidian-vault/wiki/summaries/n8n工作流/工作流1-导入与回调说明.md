# 工作流1：导入与回调说明

## 基本信息

- 工作流名称：需求开发全流程（工作流1）
- 项目：图文快印门店连锁管理系统
- n8n 默认地址：`http://localhost:5678`
- agent-app 默认地址：`http://127.0.0.1:8000`
- 工作流文件：`obsidian-vault/wiki/summaries/n8n工作流/需求开发全流程-工作流1.json`

## 导入步骤

1. 启动 n8n。
2. 打开 n8n 页面，进入 Workflows。
3. 选择 Import from File 或 Import from Clipboard。
4. 导入 `需求开发全流程-工作流1.json`。
5. 检查 Webhook 路径、HTTP Request URL 和 Code 节点。
6. 激活工作流。

## 启动工作流

启动前先双击根目录 `启动需求获取页面.bat`，确保 agent-app 已运行。

```bash
curl -X POST http://localhost:5678/webhook/requirements-workflow-1/start \
  -H "Content-Type: application/json" \
  -d '{
    "apiKey": "YOUR_BAILIAN_KEY",
    "agentBaseUrl": "http://127.0.0.1:8000",
    "acceptA5Risks": false,
    "ccbConclusion": "通过（无保留意见）"
  }'
```

可选参数：

```json
{
  "projectName": "图文快印门店连锁管理系统",
  "roundsPerStakeholder": 3,

    { "id": "operation", "name": "总部运营管理员" },
    { "id": "finance", "name": "财务人员" },
```

## CCB 审批回调

当工作流执行到 `CCB 审批等待` 节点时，使用该节点在 n8n 界面显示的 Production URL、Test URL 或等待执行详情中的 resume URL 回调。默认路径为：

```text
http://localhost:5678/webhook/requirements-workflow-1/ccb-approval
```

不同 n8n 版本对 Wait 节点的 webhook resume 地址展示方式略有差异；如果执行详情中出现 `$execution.resumeUrl` 对应的等待回调地址，以该运行时地址为准。

### 情形1：CCB 审批通过

```bash
curl -X POST http://localhost:5678/webhook/requirements-workflow-1/ccb-approval \
  -H "Content-Type: application/json" \
  -d '{
    "decision": "approved",
    "comments": "CCB 审批通过，无保留意见",
    "acceptA5Risks": false
  }'
```

### 情形2：CCB 带 A5 风险批准

```bash
curl -X POST http://localhost:5678/webhook/requirements-workflow-1/ccb-approval \
  -H "Content-Type: application/json" \
  -d '{
    "decision": "approvedWithRisk",
    "comments": "接受 A5 报告中未解决问题，带风险批准进入需求基线",
    "acceptA5Risks": true
  }'
```

### 情形3：CCB 审批不通过

```bash
curl -X POST http://localhost:5678/webhook/requirements-workflow-1/ccb-approval \
  -H "Content-Type: application/json" \
  -d '{
    "decision": "rejected",
    "comments": "当前 SRS 风险不可接受，本轮终止",
    "acceptA5Risks": false
  }'
```

### 情形4：CCB 要求退回 A4 修改 SRS

```bash
curl -X POST http://localhost:5678/webhook/requirements-workflow-1/ccb-approval \
  -H "Content-Type: application/json" \
  -d '{
    "decision": "revise",
    "comments": "请退回 A4 修订 SRS 后重新运行 A5 验证",
    "returnTo": "A4 SRS 生成"
  }'
```

## agent-app 接口关系

| 阶段 | n8n 节点 | agent-app 接口 |
|---|---|---|
| A1 | A1b 批量访谈并保存 | `POST /api/n8n/a1b-batch` |
| A2 | A2 需求质量分析 | `POST /api/a2/analyze` |
| A3 | A3 UML 建模 | `POST /api/a3/model` |
| A4 | A4 SRS 生成 | `POST /api/a4/generate` |
| A5 | A5 需求验证 | `POST /api/a5/validate` |
| A6 | A6 创建需求基线 | `POST /api/a6/create-baseline` |

工作流不会调用已删除的自动闭环接口：

- `/api/a5/auto-repair`
- `/api/a3/repair-from-a5`

## Webhook 超时说明

- n8n 默认端口为 `5678`。
- Webhook 的实际 URL 以 n8n 节点面板显示为准，Test URL 只在测试执行时可用，Production URL 需要激活工作流。
- `CCB 审批等待` 使用 Wait 节点的 webhook resume 模式。长时间暂停是否过期取决于 n8n 的执行保留和超时配置，可在 n8n 环境变量或实例设置中调整执行保留策略。
- 若固定 `ccb-approval` 路径无法恢复暂停执行，请打开该次执行详情，复制 Wait 节点提供的 resume URL 后重试 CCB 回调。
