# 技能名称：OpenAPI契约合规校验

## 操作定义

检查 OpenAPI 3.0.3 YAML 契约是否满足接口完整性、Schema 规范、错误响应、安全约束和课程设计资产归档要求。

## 输入参数

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| openapiFile | file | 是 | 待校验的 OpenAPI YAML 文件 |
| baselineSrs | file | 否 | 最新基线中的 `SRS-正式版.md` |
| moduleList | string[] | 否 | 应覆盖的业务模块 |
| requiredSecurity | string[] | 否 | 必须声明的认证或权限策略 |
| outputPath | string | 是 | 合规校验报告输出路径 |

## 执行步骤

1. 读取 YAML 文本，确认顶层包含 `openapi: 3.0.3`、`info`、`paths`、`components`。
2. 检查核心业务路径是否覆盖订单、报价、作业单、生产、库存、配送、财务、审计和报表。
3. 检查每个接口是否声明请求体、成功响应、错误响应和业务错误信息。
4. 检查 `components.schemas` 是否包含统一响应、错误响应、分页或列表结构以及关键业务 DTO。
5. 检查枚举字段是否固定声明，例如产品类型、颜色工艺、交付方式、订单状态和变更请求状态。
6. 输出缺陷清单、合规结论和修复建议。

## 输出格式

```markdown
# OpenAPI契约合规校验报告

| 检查项 | 结果 | 问题描述 | 修复建议 |
|---|---|---|---|
| 版本声明 | 通过 | - | - |

## 接口覆盖矩阵

| 模块 | 期望接口 | 实际路径 | 状态 |
|---|---|---|---|

## 结论

- [ ] 通过
- [ ] 需修复
```

## 验证步骤

- [ ] 报告明确标注 OpenAPI 版本是否为 3.0.3。
- [ ] 每个未覆盖模块都有具体缺失路径。
- [ ] 每个 Schema 问题都能定位到名称或路径。
- [ ] 校验报告不修改原 YAML 文件。
- [ ] 输出不包含 API key、令牌或本机私密路径。
