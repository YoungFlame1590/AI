## 2. 边清单 (Edges)

### 2.1 USES (Actor 使用 Component)
- A1(客户) USES C1(ORD), C5(FIN)
- A2(门店店员) USES C1(ORD), C2(QUO), C5(FIN)
- A3(门店店长) USES C2(QUO), C3(PRO), C4(DLV)
- A4(总部运营) USES C2(QUO), C4(DLV), C3(PRO)
- A5(财务人员) USES C5(FIN)
- A6(配送外协) USES C4(DLV)
- A7(系统管理员) USES C6(AUD), C7(INFRA)

### 2.2 DEPENDS_ON / PROVIDES (Component 与 Interface)
- C1(ORD) DEPENDS_ON I1(小程序API), I5(MQTT接口)
- C2(QUO) PROVIDES I6(内部报价接口)
- C3(PRO) DEPENDS_ON I5(MQTT接口)
- C4(DLV) DEPENDS_ON I6(内部路由接口)
- C5(FIN) DEPENDS_ON I2(支付网关), I3(企微Webhook), I4(税务API)
- C6(AUD) PROVIDES I6(审计拦截接口)

### 2.3 READS / WRITES (Component 与 Data)
- C1(ORD) WRITES D1(Order), D2(File)
- C2(QUO) READS D1(Order); WRITES D3(Quotation)
- C3(PRO) READS D1(Order), D3(Quotation); WRITES D4(Production)
- C4(DLV) READS D4(Production); WRITES D5(Delivery)
- C5(FIN) READS D1(Order), D5(Delivery); WRITES D6(Invoice), D7(Reconciliation)
- C6(AUD) WRITES D8(AuditLog)

### 2.4 CONSTRAINED_BY (Component 受 Constraint 约束)
- C1(ORD) CONSTRAINED_BY K1(五步闭环), K3(全量留痕)
- C2(QUO) CONSTRAINED_BY K1(五步闭环), K4(折扣互斥矩阵)
- C3(PRO) CONSTRAINED_BY K1(五步闭环), K5(信用额度阈值)
- C4(DLV) CONSTRAINED_BY K6(SLA时效优先), K1(五步闭环)
- C5(FIN) CONSTRAINED_BY K2(100%线上支付), K3(全量留痕)
- C6(AUD) CONSTRAINED_BY K3(全量留痕)

## 3. 覆盖自检
- **Actor覆盖**：7个涉众全部映射至UML Actor与SRS 2.3节，无遗漏。
- **Component覆盖**：严格对应SRS 3.1节的6大功能模块，外加基础设施模块。
- **Interface覆盖**：覆盖SRS 3.3节定义的全部4类外部接口及内部通信接口。
- **Data覆盖**：覆盖附录A数据字典核心实体及业务流转产生的关键凭证。
- **Constraint覆盖**：提取SRS 2.4节强制约束及CCB裁决的核心业务规则（如K4折扣矩阵）。
