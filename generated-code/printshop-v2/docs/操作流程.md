# Print MIS v2 操作流程

本文档记录 v2 系统的推荐操作顺序、角色边界和状态变化，后续流程调整时优先更新此文件。

## 账号

| 角色 | 账号 | 主要职责 |
|---|---|---|
| 客户 | `customer/demo123` | 下单、提交审核、确认报价、查看进度、申请发票、申请退款 |
| 门店店员 | `clerk/demo123` | 审核文件、报价、生成作业单 |
| 门店店长 | `manager/demo123` | 报价审批、排产、生产质检 |
| 总部运营 | `ops/demo123` | 生成配送/外协任务、库存维护 |
| 财务 | `finance/demo123` | 收款、开票、退款复核 |
| 配送/外协 | `courier/demo123` | 接受配送、签收 |
| 系统管理员 | `admin/demo123` | 全局查看、清空业务数据、一键演示 |

## 推荐主流程

登录后优先进入“工作台”。工作台会根据当前角色显示“我的待办任务”，点击任务即可进入订单聚合详情；聚合详情只展示当前角色需要处理或理解的区块，并根据后端返回的 `nextTasks` 显示可执行按钮。

后台仍保留订单、报价、作业单、生产、配送、财务等模块管理视图，但普通角色日常操作优先走工作台，避免在不同模块之间复制订单 ID。管理员保留全部模块作为演示和兜底维护入口。

## 操作域

| 操作域 | 覆盖角色 | 工作台职责 |
|---|---|---|
| 客户域 | 客户 | 下单、提交审核、确认报价、查看进度、申请变更、申请发票、申请退款 |
| 门店域 | 门店店员 | 文件审核、报价、生成作业单 |
| 审批域 | 门店店长 | 审批订单变更、处理门店例外 |
| 运营域 | 总部运营、配送/外协 | 排产、完工、配送创建、接单签收 |
| 财务/管理域 | 财务、系统管理员 | 收款、开票、退款处理、审计、报表、演示兜底 |

## 主流程步骤

| 步骤 | 角色 | 页面/按钮 | 前置状态 | 成功后状态 |
|---|---|---|---|---|
| 1 | 客户 | 订单 -> 新增/保存 | 无 | `SUBMITTED` |
| 2 | 客户 | 工作台/订单聚合详情 -> 提交审核 | `SUBMITTED` | `REVIEWING` |
| 3 | 店员 | 工作台/订单聚合详情 -> 生成报价 | `REVIEWING` | `QUOTED` |
| 4 | 客户 | 工作台/订单聚合详情 -> 确认报价 | 报价 `SENT/APPROVED` | 报价 `CUSTOMER_CONFIRMED` |
| 5 | 店员 | 工作台/订单聚合详情 -> 生成作业单 | `QUOTED` 且报价已确认 | `JOB_READY` |
| 6 | 总部运营 | 工作台/订单聚合详情 -> 排产 | `JOB_READY` 且库存充足 | `IN_PRODUCTION` |
| 7 | 总部运营 | 工作台/订单聚合详情 -> 完工质检 | `IN_PRODUCTION` | `PRODUCTION_DONE`，扣减库存 |
| 8 | 运营 | 工作台/订单聚合详情 -> 生成配送 | `PRODUCTION_DONE` | `DELIVERING` |
| 9 | 配送/外协 | 工作台/订单聚合详情 -> 接受配送/签收 | `DELIVERING` | `DONE` 或承运人为当前配送员 |
| 10 | 财务 | 工作台/订单聚合详情 -> 登记收款 | 报价已确认 | 订单主状态不覆盖，`paymentStatus=PAID/PARTIAL` |
| 11 | 客户 | 工作台/订单聚合详情 -> 申请发票 | 已收款且无有效发票 | 发票 `WAITING`，税号缺省为“个人无需税号” |
| 12 | 财务 | 发票 -> 开票 | 发票 `WAITING` | 发票 `ISSUED` |
| 13 | 配送/外协 | 工作台/订单聚合详情 -> 签收 | `DELIVERING` | `DONE` |
| 14 | 客户 | 工作台/订单聚合详情 -> 申请退款 | 已收款且无退款记录 | 退款记录 `REFUND_REQUESTED`，订单主状态不变 |
| 15 | 财务 | 工作台/订单聚合详情 -> 处理退款 | 退款记录 `REFUND_REQUESTED` 或已收款订单 | `REFUNDED`，`paymentStatus=REFUNDED` |

## 状态保护

系统会阻止越级操作，并在“最近响应”里给出提示：

- 未提交审核不能生成报价。
- 未生成报价或客户未确认报价不能生成作业单。
- 客户未确认报价不能登记收款。
- 未生成作业单不能排产；排产前库存不足会直接阻止排产。
- 未进入生产不能完工质检。
- 未完成生产质检不能生成配送。
- 未生成配送不能接受配送或签收。
- 未收款不能开票或退款。
- 同一订单只能存在一张有效发票；已申请或已开票后不能重复申请。
- 发票已 `ISSUED` 后不能重复开票，系统不会刷新 `issuedAt`。
- 同一订单只能存在一次退款申请或退款记录；客户申请退款不会立即改变订单终态，必须由财务处理。
- 订单已付清后不能重复收款，收款金额不能超过未收金额。

财务收款只更新付款状态，不覆盖生产/配送主流程状态，避免“收款完成”把 `IN_PRODUCTION`、`PRODUCTION_DONE` 或 `DELIVERING` 覆盖掉。

## 配送边界

配送/外协人员只处理配送相关动作：

- 可见：待接单或已分配给自己的配送订单。
- 可执行：接受配送、签收。
- 不可执行：生成配送任务、上传订单文件、查看订单文件、维护非配送字段。

## 库存扣减

系统默认保留 A4 纸、铜版纸、彩色墨粉/墨水和装订耗材。排产前系统会根据订单产品、颜色工艺、页数和份数估算所需物料；任一物料不足时直接拒绝排产并提示缺口。生产任务点击“完工质检通过”后，系统再次校验并扣减库存，避免负库存。

## 数值与报表

- 订单页数必须是 `1-5000` 的整数，份数必须是 `1-10000` 的整数；页面使用整数步进输入，后端作为最终校验边界。
- 报表从当前订单、付款、发票、生产和库存实时聚合。完成订单或登记收款后，`/api/reports` 会展示订单漏斗、运营摘要、门店汇总和财务摘要，不再依赖预置样例数据。

## 清空数据

管理员登录后点击顶栏“清空业务数据”，会删除订单、报价、作业单、生产、配送、财务和审计数据，但保留基础门店、七类账号和默认库存。
