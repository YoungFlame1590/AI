# 技能名称：CIA变更影响分析

## 操作定义

基于变更请求 CR、当前需求基线、SRS、设计资产和代码模块，执行九维度变更影响分析，输出 CIA 报告。

## 输入参数

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| crFile | file | 是 | 变更请求文档 |
| baselineDir | directory | 是 | 当前基线目录 `BL-YYYYMMDD-NN` |
| srsFile | file | 是 | 当前正式 SRS |
| designFiles | file[] | 是 | MDS、DTS、TLCD、OpenAPI、ADR 等设计资产 |
| codeRoot | directory | 否 | 受影响代码根目录 |
| outputPath | string | 是 | CIA 报告输出路径 |

## 执行步骤

1. 读取 CR，提取变更主题、业务原因、期望行为和影响对象。
2. 从基线和 SRS 中定位受影响需求，标记新增、修改、删除或未变更。
3. 检查架构、模块、契约、数据和代码层是否需要同步调整。
4. 分析兼容性、测试范围、上线风险和是否需要拆分多个 CR。
5. 给出变更规模评估和 CCB 审批建议。
6. 输出九维度 CIA 报告。

## 输出格式

```markdown
# CIA 变更影响分析报告

| 维度 | 影响结论 | 受影响对象 | 风险等级 | 建议动作 |
|---|---|---|---|---|
| 需求维度 | ... | ... | 中 | ... |

## 受影响需求

| 需求ID | 变更类型 | v1描述 | v2描述 | 依据 |
|---|---|---|---|---|

## CCB 建议

- 变更规模：小/中/大
- 建议：批准/有条件批准/退回澄清
```

## 验证步骤

- [ ] 九个维度均有结论，不得缺项。
- [ ] 每个受影响需求都能追溯到 CR 或基线来源。
- [ ] 明确是否需要更新 OpenAPI、TLCD、代码和测试。
- [ ] 明确给出 CCB 建议和变更规模。
- [ ] 输出不包含 API key、令牌或本机私密路径。
