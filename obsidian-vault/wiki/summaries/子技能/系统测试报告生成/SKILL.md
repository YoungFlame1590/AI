# 技能名称：系统测试报告生成

## 操作定义

汇总自动化测试、端到端场景、回归校验和知识库检查结果，生成课程格式的系统软件测试报告。

## 输入参数

| 参数名 | 类型 | 必填 | 说明 |
|---|---|---|---|
| testTarget | string | 是 | 被测系统名称或目录，例如 `generated-code/printshop-v2` |
| commandResults | object[] | 是 | Maven、JS、CRR、compile.js 等命令执行结果 |
| e2eResult | object | 否 | HTTP 端到端测试步骤结果 |
| testAccounts | object[] | 否 | 测试账号与角色列表 |
| riskNotes | string[] | 否 | 遗留风险说明 |
| outputPath | string | 是 | 测试报告输出路径 |

## 执行步骤

1. 读取所有命令结果，统计通过、失败、耗时和失败原因。
2. 汇总端到端场景步骤，确认是否覆盖下单、审核、报价、生产、配送、财务、审计和报表。
3. 将测试范围映射到功能测试、权限测试、变更测试、边界测试、回归测试、前端测试和知识库测试。
4. 生成缺陷清单，对失败项标注严重程度和处理建议。
5. 生成最终结论：通过、带风险通过或不通过。
6. 输出 Markdown 测试报告。

## 输出格式

```markdown
# 系统软件测试报告

## 测试结论

vX 系统软件测试通过/未通过。

## 自动化执行结果

| 步骤 | 结果 | 耗时ms | 错误 |
|---|---|---:|---|

## 覆盖矩阵

| 测试类别 | 覆盖内容 | 结果 |
|---|---|---|

## 缺陷清单

| 编号 | 严重程度 | 描述 | 状态 |
|---|---|---|---|
```

## 验证步骤

- [ ] 报告包含测试结论、环境、账号、覆盖矩阵、自动化结果、缺陷清单和遗留风险。
- [ ] 所有失败命令必须进入缺陷清单。
- [ ] 端到端结果必须能追溯到具体业务步骤。
- [ ] 通过结论只能在阻塞失败数为 0 时给出。
- [ ] 输出不包含 API key、令牌或本机私密路径。
