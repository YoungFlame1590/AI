## 工程形态

- Java 17 + Spring Boot 3。
- MySQL 8 + Spring Data JPA + Flyway。
- Spring Security Basic Auth，提供七类演示账号。
- 静态原生 HTML/CSS/JS 前端由 Spring Boot 托管。
- 文件上传保存到本地 `uploads/`，数据库保存文件元数据。

## 分层结构

- `common`：统一响应、异常处理。
- `infra`：统计、追踪和后续外部系统防腐层入口。
- `mis/domain`：JPA 实体。
- `mis/repository`：Spring Data JPA 仓储。
- `mis/service`：业务编排、状态流转、审计记录。
- `mis/controller`：REST API 入口。
- `mis/security`：Basic Auth 登录与角色身份加载。

## 前端业务系统

访问入口：

```text
http://127.0.0.1:8080/
```

前端提供左侧模块导航、顶部当前用户/门店/角色、列表、详情、创建、编辑、删除和状态流转动作。订单详情与工作台数据覆盖基础信息、文件、报价、作业单、生产进度、配送、财务和审计时间线。

## 运行命令

```powershell
cd generated-code\printshop-v1
docker compose up -d mysql
mvn spring-boot:run
```

MySQL 容器映射到 `127.0.0.1:13306`，避免与本机已有 MySQL 的 `3306` 端口冲突。

Docker 镜像不可用时可使用 H2 演示 profile：

```powershell
mvn spring-boot:run -Dspring-boot.run.profiles=demo
```

## v1 已知边界

- Basic Auth 和演示密码仅用于本地课程演示。
- 支付、税务、短信/企微和设备接口暂不接真实第三方。
- 文件上传使用本地目录，后续版本可替换为对象存储。
- v1 保持模块化单体，不实现微服务拆分和分布式事务。
