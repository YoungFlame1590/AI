# 代码层资产清单

## 资产清单

| 资产 | 路径 | 说明 |
|---|---|---|
| v1 业务工程 | `generated-code/printshop-v1/` | Spring Boot 3 + Java 17 的 Print MIS 初版业务系统 |
| v2 业务工程 | `generated-code/printshop-v2/` | 在 v1 基础上增加订单变更冻结、变更请求审批和系统测试 |
| v1 漂移检测脚本 | `generated-code/printshop-v1/scripts/verify-design-drift.js` | 生成 RCR 与模块设计质量校验 |
| v2 变更回归脚本 | `generated-code/printshop-v2/scripts/verify-change-regression.js` | 校验 CR-001 是否实现且不破坏清洁模块 |
| v2 系统测试脚本 | `generated-code/printshop-v2/scripts/run-system-tests.ps1` | 汇总 Maven、JS、CRR、知识库和 E2E 测试 |
| v2 端到端测试脚本 | `generated-code/printshop-v2/scripts/e2e-smoke.ps1` | 覆盖下单、审核、报价、变更冻结、审批、生产、配送、财务和报表 |

## 复用价值

- 提供准生产版 Print MIS 的分层工程样例。
- 展示 Controller、Service、Repository、Domain、Security、Static Frontend 的组织方式。
- 提供角色权限、审计日志、统计接口、数据清理接口和端到端测试的可复用结构。
- 展示从 v1 到 v2 的增量变更方式，而不是直接覆盖旧版本。

## 迁移指南

1. 保留 Maven、Spring Boot、Flyway、Security、JPA 的基础结构。
2. 替换 `com.printshop` 包名、业务模块名称和数据库表结构。
3. 将 Print MIS 的订单、报价、排产、配送、财务模块替换为新项目领域模块。
4. 保留 `/stats`、审计日志、系统测试脚本和漂移检测脚本的结构，调整模块清单和接口路径。
5. 新项目若不需要 v2 变更闭环，可先只迁移 v1；若需要需求变更管理，再复制 v2 的 CR 实现方式。

## 复用边界

- 不直接复用真实支付、税务、短信或外部设备接口，因为当前工程使用模拟适配。
- 不直接复用本项目的业务枚举，例如产品类型、颜色工艺、交付方式。
- 不复用 `target/`、`uploads/`、本地数据库文件或运行日志。
