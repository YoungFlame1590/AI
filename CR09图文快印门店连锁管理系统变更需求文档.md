# 变更需求文档（CR）

---

**基线版本**：PRINTCHAIN-SRS-V2.0（BL-20260501-01）  
**变更提出日期**：2026年7月  
**变更提出人**：总部运营经理 + 店长  
**文档状态**：待CCB审批

## 变更概述

本文档记录图文快印门店连锁管理系统在v1版本上线试运行后，根据门店运营数据和客户需求变化提出的4项变更需求。其中1项为重大变更（涉及在线设计编辑器与模板市场），3项为轻微变更（功能增强与业务规则优化）。

---

## CR-01：在线设计编辑器与模板市场（重大变更）

| 属性 | 内容 |
|------|------|
| **变更编号** | CR-PRINTCHAIN-001 |
| **变更来源** | 总部运营经理 + 店长（客户自助服务趋势） |
| **变更优先级** | **高** |
| **变更类型** | 架构变更 + 新功能模块 |
| **变更规模** | **重大**（新增在线编辑器、模板系统、渲染引擎、文件管理） |

### 变更描述

**当前行为**（v1）：客户到店提交设计文件（U盘拷贝、微信/QQ发送、邮件附件），店员下载后预览确认。客户如需修改设计，需自行用专业软件（Photoshop、Illustrator等）修改后重新提交，往返多次。系统仅做文件接收和存储，不提供任何在线编辑或设计辅助功能。

**期望行为**（v2）：新增"在线设计编辑器"模块——（1）所见即所得编辑器：基于Web的轻量级设计工具（Canvas/SVG渲染），客户无需安装任何软件，在浏览器中即可进行文字排版、图片替换、二维码生成、Logo添加等常见操作；（2）模板市场：预置各行业常用模板（名片、宣传单、海报、展架、折页、画册封面），客户选择模板后替换文字和图片即可快速完成设计；（3）在线下单：设计完成后一键下单到门店，自动填写印刷规格参数（尺寸由模板自动确定、纸张和工艺可选）；（4）设计文件版本管理：每次修改自动保存版本，客户可回溯任意历史版本。

**差异分析**：

| 维度 | v1（当前） | v2（期望） | 影响 |
|------|-----------|-----------|------|
| 设计能力 | 无，依赖客户自带设计文件 | 在线轻量设计 + 模板 | 新增设计编辑器前端应用 |
| 模板系统 | 无 | 模板市场（总部创建+门店共享） | 新增模板管理后台 |
| 文件流转 | 文件上传→下载→手动检查 | 在线设计→规格自动填充→一键下单 | 订单创建流程重构 |
| 客户协作 | 反复线下沟通修改 | 在线实时编辑+版本管理 | 新增协作功能 |
| 存储 | 仅最终文件存储 | 设计过程文件+版本存储 | 存储需求增加 |

### 变更理由

1. **客户痛点**：小型企业和个体商户是图文快印的主要客户群，他们往往不具备专业设计能力——"我知道要印什么，但我不会做设计文件"。当前模式迫使他们额外花钱找设计师，增加了客户的综合成本和时间。
2. **差异化竞争**：区域内仅有1家竞争对手提供了基础的在线模板功能（仅限名片），甲方率先上线完整的在线设计编辑器+多品类模板可形成显著的差异化优势。
3. **设计效率**：店员当前约20%的工作时间用于帮客户做简单设计调整（加文字、换Logo、调尺寸），编辑器可使客户自行完成这些操作，释放店员产能。
4. **模板变现**：模板市场可设置免费模板（基础款）和付费模板（设计师精品款），开辟新的营收来源。

### 影响评估（初步）

- **架构**：新增在线编辑器前端应用（Web Canvas/SVG技术栈）+ 模板管理服务 + 渲染引擎（将设计稿转换为印刷规格）+ 版本管理服务。
- **模块影响**：门店收银模块（在线下单入口+规格自动填充）、文件管理模块（版本管理）、新增模板管理模块（总部创建/审核/发布模板）。
- **技术风险**：在线编辑器的性能（Canvas渲染流畅度）和浏览器兼容性是关键风险点；印刷文件生成（将在线设计导出为300dpi CMYK PDF）的技术实现难度较高。
- **接口契约**：新增编辑器相关API（模板获取、设计保存、设计提交下单）、模板管理API。

## CR-02：耗材智能补货与自动采购（轻微变更）

| 属性 | 内容 |
|------|------|
| **变更编号** | CR-PRINTCHAIN-002 |
| **变更来源** | 采购 + 店长（库存管理痛点） |
| **变更优先级** | **中** |
| **变更类型** | 业务规则增强 |
| **变更规模** | **轻微**（补货算法 + 采购流程自动化） |

### 变更描述

**当前行为**（v1）：各门店耗材库存低于安全库存时自动通知店长+采购，采购员人工汇总各门店需求后生成统一采购单。

**期望行为**（v2）：升级为智能补货——（1）动态安全库存：基于过去90天的耗材消耗速度和供应商交货提前期，自动计算各门店每种耗材的动态安全库存量（替代当前的静态手动设定值）；（2）自动采购建议：当总部汇总各门店需求后，系统自动生成最优采购方案——合并相同品类、考虑供应商最小起订量和批量折扣阶梯、推荐最优采购数量以平衡采购成本和存储成本；（3）采购预测：基于历史同期（去年同月、上月）和门店增长趋势，预测未来30天的耗材需求，生成预采购计划供采购员参考。

**差异**：v1静态安全库存+人工汇总采购；v2动态安全库存+智能采购方案+需求预测。

### 变更理由

1. **库存精准度**：静态安全库存导致部分门店"过度囤货"（占用资金和存储空间）而另一些门店"频繁缺货"（影响门店运营），动态计算可平衡库存水平。
2. **采购成本**：批量折扣阶梯（如铜版纸100令以下单价X、100-500令单价0.95X、500令以上单价0.9X）的利用需要精确计算最优采购量，人工难以做到。
3. **减少缺货**：预测模型可提前预判需求高峰（如开学季对宣传单页的需求激增），提前备货。

## CR-03：门店服务质量评价与客户反馈（轻微变更）

| 属性 | 内容 |
|------|------|
| **变更编号** | CR-PRINTCHAIN-003 |
| **变更来源** | 总部运营经理（门店管理精细化） |
| **变更优先级** | **中** |
| **变更类型** | 新功能 |
| **变更规模** | **轻微**（新增评价模块 + 统计分析） |

### 变更描述

**当前行为**（v1）：无系统化的客户反馈收集和服务质量评价机制。客户不满通常通过投诉电话反馈，无系统记录和分析。

**期望行为**（v2）：新增"客户评价"模块——（1）订单完成后自动推送评价邀请（小程序消息/短信），客户对本次服务进行星级评分（1-5星）和文字评价，评价维度：印刷质量、交付准时性、店员服务态度、性价比；（2）差评自动工单：评分≤2星的评价自动生成"客诉工单"，推送给店长处理（24小时内回复客户），处理结果记录留痕；（3）门店服务质量排行：总部看板展示各门店的综合评分趋势、差评率、客诉处理时效等指标，纳入门店绩效考核；（4）客户回访提醒：对30天未消费的老客户自动提醒店长进行回访关怀。

**差异**：v1无评价体系；v2完整的评价→工单→改进闭环。

### 变更理由

1. **服务质量管控**：连锁门店的核心挑战是保持各门店服务质量一致，数据化的评价体系是客观衡量服务质量的基础。
2. **客户留存**：及时处理客户不满可将客户流失率降低50%以上；老客户回访可提升复购率。
3. **门店激励**：服务质量排行可与店长绩效挂钩，形成正向竞争。

## CR-04：第三方配送对接（轻微变更）

| 属性 | 内容 |
|------|------|
| **变更编号** | CR-PRINTCHAIN-004 |
| **变更来源** | 店长 + 客户（交付便捷性） |
| **变更优先级** | **低** |
| **变更类型** | 外部对接 |
| **变更规模** | **轻微**（对接即时配送/快递API） |

### 变更描述

**当前行为**（v1）：成品交付方式为门店自取，系统不做配送管理。客户如需配送，由店员手动联系快递公司或跑腿服务，费用和单号不纳入系统管理。

**期望行为**（v2）：订单完成时提供配送选项——（1）即时配送：对接美团跑腿/闪送/达达API，适合单店周边5公里内的急件配送（1小时内送达）；（2）快递配送：对接顺丰/圆通/中通API，适合跨区域配送；（3）配送费自动计算：根据取件地址、送达地址、包裹重量自动估算各渠道配送费，客户选择后配送费加入订单；（4）配送状态追踪：客户可在小程序端实时追踪配送状态。

**差异**：v1无配送管理；v2即时配送+快递+费用估算+状态追踪。

### 变更理由

1. **客户需求**：约30%的客户询问配送服务（尤其是企业客户批量印制后需要送上门），当前无标准化配送流程导致体验割裂。
2. **服务延伸**：配送能力的补齐使门店服务从"到店取件"扩展到"门到门"，提升了服务完整性和竞争力。
3. **实施成熟**：即时配送和快递API已非常成熟（顺丰/闪送均有标准开发者接口），集成难度低。
