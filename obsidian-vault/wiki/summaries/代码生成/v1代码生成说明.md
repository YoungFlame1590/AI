## 分层结构

- `infra`：审计、统计、状态机、支付/税务/机台防腐层占位适配器。
- `repository`：各业务模块内存仓储。
- `application`：应用服务，承载 v1 业务规则。
- `controller`：REST API 入口，路径与 OpenAPI 契约一致。

## 七类角色业务工作台

v1 工程新增由 Spring Boot 静态资源目录托管的前端页面：

```text
generated-code/printshop-v1/src/main/resources/static/
```

访问入口：

```text
http://127.0.0.1:8080/
```

页面能力：

- 支持客户、门店店员、门店店长、总部运营管理员、财务人员、配送/外协人员和系统管理员七类角色切换。
- 每个角色展示专属指标、待办、订单队列、详情动作、审计日志和 `/stats` 摘要。
- 新增 `GET /api/v1/roles`、`GET /api/v1/workbench/{roleId}`、`POST /api/v1/workbench/actions`、`POST /api/v1/workbench/reset` 作为角色工作台 Facade API。

## v1 已知简化项

- Repository 使用内存存储，重启后数据清空。
- 角色工作台使用内存种子数据，不实现真实登录、权限或数据库持久化。
- `/stats` 使用内存统计，重启后清零。
- 支付、税务、企微和机台扫码接口暂用 INFRA 防腐层占位适配器。
- v1 不实现真实数据库事务边界之外的分布式事务、消息队列或外部系统联调。
