## 工程形态

- Java 17 + Spring Boot 3。
- MySQL 8 + Spring Data JPA + Flyway。
- Spring Security Basic Auth，提供七类演示账号。
- 静态原生 HTML/CSS/JS 前端由 Spring Boot 托管。
- 文件上传保存到本地 `uploads/`，数据库保存文件元数据。

## 后端分层结构

- `common`：统一响应、异常处理。
- `infra`：统计、追踪和后续外部系统防腐层入口。
- `mis/domain`：JPA 实体。
- `mis/repository`：Spring Data JPA 仓储。
- `mis/identity`：登录、当前用户、用户与门店查询。
- `mis/order`：订单、订单文件、订单权限策略、固定选项和自动计价。
- `mis/maintenance`：管理员清空业务数据，保留基础门店与七类账号。
- `mis/quotation`：报价生成、折扣和审批。
- `mis/job`：作业单。
- `mis/production`：生产排程、完工和质检。
- `mis/inventory`：默认库存、库存调整、生产扣减和低库存预警。
- `mis/delivery`：配送/外协任务、签收和异常记录。
- `mis/finance`：发票、收款和退款。
- `mis/audit`：审计日志。
- `mis/reporting`：报表汇总。
- `mis/dashboard`：当前用户工作台聚合。
- `mis/security`：Basic Auth 登录与角色身份加载。

## 前端分层结构

前端继续由 Spring Boot 静态托管，不引入 npm/Vite/Vue/React。入口为 `index.html` 加载 `type="module"` 的 `js/main.js`：

- `static/js/config.js`：模块配置、角色菜单、订单选项和价目表。
- `static/js/state.js`：登录用户、当前模块、选中记录和编辑状态。
- `static/js/api.js`：统一 `fetch`、鉴权 Header 和错误处理。
- `static/js/orders.js`：订单下拉框、字段权限和金额预估。
- `static/js/render.js`：导航、表格、详情表单、指标和时间线渲染。
- `static/js/main.js`：事件绑定与启动入口。

## 前端业务系统

访问入口：

```text
http://127.0.0.1:8080/
```

前端提供左侧模块导航、顶部当前用户/门店/角色、列表、详情、创建、编辑、删除和状态流转动作。订单详情与工作台数据覆盖基础信息、文件、报价、作业单、生产进度、配送、财务和审计时间线。

订单模块已按角色和门店隔离：客户仅能访问本人订单，门店店员/店长仅能访问本门店订单，配送/外协仅能访问本人或待接单配送相关订单。订单产品类型、颜色/工艺、交付方式和优先级使用固定下拉选项；金额由后端固定价目表计算，前端仅展示预估值。

推荐操作顺序维护在 `generated-code/printshop-v1/docs/操作流程.md`，用于说明角色、按钮、前置状态、成功后状态和越级提示。

订单详情提供快捷流转动作：提交审核、生成报价、生成作业单、排产任务、生成配送、接受配送、登记收款、生成退款、生成发票，以及管理员专用的一键跑完整链路。快捷动作会自动复用当前订单上下文，避免在多个模块间手工复制订单 ID、报价 ID 和作业单 ID。

配送员只负责接受待接配送订单和签收配送任务，不显示订单文件上传/查看，也不能生成配送任务。签收后订单状态同步为 `DONE`。

财务登记收款只更新付款状态为 `PAID` 或 `PARTIAL`，不覆盖订单主流程状态；客户、财务或管理员可从订单详情生成退款记录，订单状态与付款状态同步为 `REFUNDED`。发票缺省税号会自动填充为“个人无需税号”。

系统提供默认库存，生产任务完工质检通过后会按订单页数、份数和工艺扣减纸张、墨粉/墨水和装订耗材。

管理员顶栏提供“清空业务数据”按钮，对应 `DELETE /api/admin/business-data`。该操作删除业务单据、财务和审计数据，但保留基础门店、七类演示账号与默认库存。

## 运行命令

```powershell
cd generated-code\printshop-v1
docker compose up -d mysql
mvn spring-boot:run
```

MySQL 容器映射到 `127.0.0.1:13306`，避免与本机已有 MySQL 的 `3306` 端口冲突。

Docker 镜像不可用时可使用 H2 演示 profile：

```powershell
mvn spring-boot:run -Dspring-boot.run.profiles=demo
```
