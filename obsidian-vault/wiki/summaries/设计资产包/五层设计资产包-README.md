# 五层设计资产包 README

## 资产包用途

本资产包面向“图文快印门店连锁管理系统”的后续复用，整理从需求工程、架构设计、AI 代码生成、逆向校验到需求变更管理的完整课程产物。资产包采用“索引 + 复用指南”方式归档，不复制大体量源码，避免源码与知识库双份维护。

适用场景：

- 新项目快速搭建 Obsidian 知识库与需求工程流水线。
- 复用 Print MIS 的分层架构、OpenAPI 契约和测试策略。
- 将 A1-A6、n8n、设计约束、代码生成、CR/CIA/CRR 流程迁移到其他课程项目。

## 五层资产总览

| 层级 | 核心资产 | 复用价值 | 主要替换内容 |
|---|---|---|---|
| 代码层 | v1/v2 Spring Boot 工程、测试脚本、漂移检测脚本 | 快速复用分层代码、角色工作台、CR-001 变更实现 | 包名、业务模块、数据库字段、页面文案 |
| 契约层 | OpenAPI v1/v2、TLCD、约束提示词 | 约束 AI 代码生成，统一接口和错误响应 | 接口路径、Schema、枚举、权限规则 |
| 知识图谱层 | Obsidian 四层知识库、SRS、UML、ADR、基线、RTM | 保留需求到设计到测试的追溯链 | 项目名称、涉众、需求条目、基线编号 |
| 子技能层 | 5 个 `SKILL.md` | 将常用工程操作封装为可复用流程 | 输入路径、输出路径、判定阈值 |
| 提示词模板层 | A1-A6、设计阶段、n8n、变更管理提示词 | 快速复刻智能体与工作流提示词 | 项目领域、业务术语、模型参数 |

## 复用流程

1. 复制知识库四层结构与 `compile.js`，替换项目名称和涉众清单。
2. 使用提示词模板层创建 A1-A6 智能体，生成 raw notes、A2 报告、UML、SRS、A5 验证报告和 A6 基线。
3. 基于知识图谱层产物运行架构选型，生成 ASD、MDS、DTS、ADR-001 至 ADR-004。
4. 使用契约层的 TLCD 与 OpenAPI 模板约束 AI 生成代码。
5. 参考代码层的 v1/v2 工程组织业务代码，并运行 RCR/CRR 与系统测试。
6. 遇到需求变更时，按 CR → CIA → 契约更新 → 代码变更 → CRR → ADR-005 → 新基线流程推进。

## 可直接复用内容

- Obsidian 四层目录规范、`compile.js` 检查策略。
- A1-A6 智能体分工、n8n 工作流人工 CCB 审批思路。
- Spring Boot 分层结构、审计日志、角色权限、统计接口和系统测试脚本组织方式。
- RCR/CRR 报告格式、四类漂移检测、四维度质量校验格式。
- 5 个 Reasonix 子技能的输入、输出和自检模板。

## 必须替换内容

- 项目名称、涉众角色、业务对象和业务流程。
- SRS 中的 REQ 编号、验收标准、数据字典和非功能指标。
- OpenAPI 路径、Schema 字段、枚举值和权限规则。
- 代码包名、模块名称、数据库表结构和前端文案。
- 基线编号、CR 编号、CCB 审批结论和变更影响范围。
