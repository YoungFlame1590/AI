# 软件需求规格说明书

**项目名称**：图文快印门店连锁管理系统
**文档版本**：v1.0-draft
**生成日期**：2026-06-18

## 1. 引言

### 1.1 目的
本文档旨在定义图文快印门店连锁管理系统的软件需求规格，明确系统功能、性能、接口及数据约束。本文档作为系统设计、编码实现、测试验证与项目验收的唯一基准，确保所有开发活动可追溯、可量化、可验证。

### 1.2 范围
本系统覆盖图文快印连锁门店的核心业务链路，包含订单接单、文件合规校验、阶梯价自动计算、折扣审批、排产调度、信用额度控制、跨店路由与配送执行、电子发票开具、日结对账、权限管控与全链路审计。系统服务对象包含客户、门店店员、门店店长、总部运营管理员、财务人员、配送/外协人员及系统管理员。

## 2. 总体描述

### 2.1 产品视角
本系统为连锁型SaaS平台，包含微信小程序/扫码客户端、门店PC操作端与工位看板、总部运营管理后台、财务对账端、配送人员移动端及系统管理员配置端。系统需与第三方支付渠道（微信/支付宝/对公）、企业微信消息推送服务、电子发票税务接口及门店机台扫码设备对接。

### 2.2 产品功能概述
系统核心能力划分为六大模块：订单接单与文件校验、报价与折扣审批、排产调度与信用开工、跨店派单与配送执行、发票开具与日结对账、权限配置与审计日志。各模块严格遵循状态机流转，支持可配置的业务分支（如发票触发节点、路由权重），并强制记录全量操作轨迹。

### 2.3 用户特征
| 角色 | 核心特征与操作频次 |
|---|---|
| 客户 | 学生/行政/商户，周均下单3至5次，关注报价锁定、进度可见、文件安全与发票自动触达。 |
| 门店店员 | 日均处理40至80单，需3分钟内完成参数核对与报价录入，依赖系统自动防错与状态同步。 |
| 门店店长 | 负责排产调度、折扣特批、异常接管，要求保留应急开关与手动优先级调整权限，每日10分钟内掌握积压与逾期数据。 |
| 总部运营管理员 | 管理30余家门店，制定价格/活动/路由规则，审批超限折扣，监控五步闭环完成率与线上收款占比。 |
| 财务人员 | 处理日结对账与发票开具，要求账实一致、差异精准定位至订单与操作人，日结对账耗时压至30分钟内。 |
| 配送/外协人员 | 日均接8至12单跨店调货，依赖首屏关键信息展示与异常在线报备，要求审批超时20分钟自动放行。 |
| 系统管理员 | 管控账号权限、路由权重与基础配置，要求变更100%留痕、高危操作强阻断、故障15分钟内定位。 |

### 2.4 约束条件
1. 强制五步闭环流转，严禁跳步（除应急开关触发外）。
2. 100%线上支付归集，线下收款视为违规。
3. 所有人工干预、改价、应急放行必须强制记录操作人、精确到秒时间戳及业务原因。
4. 系统可用性在业务高峰期不低于99.9%。

### 2.5 假设与依赖
1. 假设门店网络带宽稳定，支持大文件上传与实时状态同步。
2. 依赖第三方支付平台提供实时交易流水查询接口。
3. 依赖企业微信API实现待办推送与审批流路由。
4. 依赖门店机台支持二维码扫码拦截与状态上报。

## 3. 具体需求

### 3.1 功能需求

#### 模块：订单接单与文件校验 (ORD)

**REQ-ORD-001**
- 描述：客户通过微信小程序或到店扫码提交订单与设计文件，系统执行格式与容量校验。
- 输入：订单基础参数、设计文件（PDF/AI/CDR等）。
- 输出：临时单号、校验通过/拦截结果、缩略预览。
- 前置条件：客户完成身份登录或扫码绑定。
- 验收标准：单文件上限100MB、页数上限80页；超限直接标红拦截并提示“拆分或转PDF”；上传拦截响应＜2秒，预览加载＜3秒；文件全程加密存储。
- 来源涉众：客户、门店店员
- 优先级：高

**REQ-ORD-002**
- 描述：订单满足合规、支付与质检条件后，自动流转至生产状态。
- 输入：支付成功状态、前台质检结果。
- 输出：订单状态变更（待排产/生产中）。
- 前置条件：文件可正常打开且页数/幅面在常规设备范围内；系统报价与到店实价一致；客户已全额支付；门店前台完成初步质检。
- 验收标准：普通订单提交后30分钟内流转，急件≤1小时内流转至“生产中”；条件不满足时订单卡在“待审核/待付款”环节。
- 来源涉众：客户、门店店员
- 优先级：高

**REQ-ORD-003**
- 描述：订单状态变更时向客户推送进度通知。
- 输入：节点状态事件。
- 输出：客户端消息推送。
- 前置条件：订单进入制作环节。
- 验收标准：仅保留“门店已接单→制作中→已备好（附取件码）”三步状态；通知延迟≤2分钟；取件前至少提前15分钟明确提醒；支持常用规格一键复购。
- 来源涉众：客户
- 优先级：高

#### 模块：报价与折扣审批 (QUO)

**REQ-QUO-001**
- 描述：系统根据录入参数自动计算阶梯报价。
- 输入：纸张克重/材质、黑白/彩色、总页数、工艺（覆膜/装订）、数量。
- 输出：标准报价明细。
- 前置条件：订单参数完整录入。
- 验收标准：常规订单3分钟内自动带出报价；线上明细与到店实付分毫不差；价格推送延迟＜30秒。
- 来源涉众：门店店员、客户
- 优先级：高

**REQ-QUO-002**
- 描述：折扣申请触发分级审批路由。
- 输入：折扣比例或减免金额。
- 输出：审批通过/驳回/升级结果。
- 前置条件：客户申请折扣或店员发起加急附加费调整。
- 验收标准：单笔减免≤15元或实收≥9折，店长直批（响应≤30秒）；超限或实收＜9折，流转总部线上审批；审批超时2小时自动升级至区域经理，再超时2小时直接升级至总部运营负责人并抄送财务备案。
- 来源涉众：门店店长、总部运营管理员、财务人员
- 优先级：高

**REQ-QUO-003**
- 描述：制作过程中客户临时增加附加服务或触发加价。
- 输入：附加项勾选、价格变动值。
- 输出：更新后总价与剩余预计时间。
- 前置条件：订单处于“制作中”状态。
- 验收标准：总价与剩余时间实时刷新；单笔加价＞5元需二次确认弹窗；未勾选“同意”前不扣款、不进下一环节。
- 来源涉众：客户
- 优先级：中

#### 模块：排产调度与生产流转 (PRO)

**REQ-PRO-001**
- 描述：系统根据订单类型与额度控制排产指令下发。
- 输入：订单类型（全款/信用）、授信额度、财务到账状态。
- 输出：机台排产指令或拦截提示。
- 前置条件：订单通过价格校验。
- 验收标准：信用单单笔≤3000元且月累计≤1万元，额度触达80%系统提示风险，100%强制拦截转主管审批；全款单需财务确认到账后解锁；触发应急开关强制下发需记录应急原因与审计水印。
- 来源涉众：门店店长、财务人员
- 优先级：高

**REQ-PRO-002**
- 描述：改价或工艺变更审批期间冻结生产作业。
- 输入：审批中状态标识。
- 输出：机台扫码拦截指令、SLA暂停标记。
- 前置条件：店长或总部发起折扣/工艺变更审批。
- 验收标准：车间必须暂停该单扫码作业，严禁转交空闲同事；SLA倒计时同步暂停并标记“待审批冻结期”；解冻后由值班组长手动调回原机台优先插队；审批响应压至30分钟内，冻结期不计入逾期考核。
- 来源涉众：门店店长、总部运营管理员
- 优先级：高

**REQ-PRO-003**
- 描述：突发停工或断料触发人工兜底干预。
- 输入：异常原因码、改派/拆单方案。
- 输出：干预执行结果与对比记录。
- 前置条件：设备故障或核心耗材短缺。
- 验收标准：后台人工兜底最多允许2步（强制改派至指定门店、拆单重审或手工核销）；严禁第3步随意修改单价；强制记录操作人、精确到秒时间戳及业务原因；异常月均发生15至20起需在4小时内闭环。
- 来源涉众：总部运营管理员、系统管理员
- 优先级：高

#### 模块：跨店派单与配送执行 (LOG)

**REQ-LOG-001**
- 描述：跨店调货路由引擎执行派单决策。
- 输入：SLA承诺时效、门店库存、设备负荷、业务资质。
- 输出：目标派单门店。
- 前置条件：触发跨店调度或集中加工。
- 验收标准：以SLA承诺时效为第一阈值；若两家均能按期交付，优先派给库存充足店；资质或设备离线硬性拦截重路由；设备负荷≤60%作为动态均衡次优指标；跨店履约准时率≥95%。
- 来源涉众：总部运营管理员、系统管理员
- 优先级：高

**REQ-LOG-002**
- 描述：配送任务下发与途中异常在线处理。
- 输入：任务清单、异常照片/改址申请。
- 输出：签收状态、审批流转结果。
- 前置条件：配送/外协人员接单。
- 验收标准：首屏固定显示取送地址与对接人电话、最晚送达/完工截止时间、物品明细与外协工艺要求；派单后3分钟内清单可查；改址/加急审批超时20分钟未批复系统自动放行并标黄预警；签收必扫条码或电子签名。
- 来源涉众：配送/外协人员、门店店长
- 优先级：高

#### 模块：发票开具与日结对账 (FIN)

**REQ-FIN-001**
- 描述：电子发票生成触发与逆向红冲控制。
- 输入：取件确认状态/支付流水、退款申请。
- 输出：电子发票PDF、红冲凭证。
- 前置条件：业务规则配置选定触发节点（交付后开票或付款预开）。
- 验收标准：配置为交付后开票时，扫码取件确认后1小时内推送至指定邮箱；配置为付款预开时，退款触发原发票冻结，红冲必须绑定退款流水号并经财务审批；自动开票上限单笔≤500元且非企业月结账户；跨店分摊或单笔＞3000元拦截转人工复核。
- 来源涉众：财务人员、客户
- 优先级：高

**REQ-FIN-002**
- 描述：每日闭店自动触发对账并生成日报。
- 输入：当日订单流水、第三方支付渠道账单。
- 输出：日结对账报表。
- 前置条件：营业时间结束。
- 验收标准：报表含订单号、操作人、支付方式及渠道单号、应收/实收/手续费、发票状态、退款标记；差异≤100元自动计入营业外收支平账；差异＞100元标红推送店长复核；日结对账耗时压至30分钟内。
- 来源涉众：财务人员、门店店员
- 优先级：高

**REQ-FIN-003**
- 描述：计价延迟前端交互与尾差自动处理。
- 输入：计价明细、实付金额、网络延迟时长。
- 输出：弹窗提示、平账流水。
- 前置条件：客户进入结算页面。
- 验收标准：价格推送超2分钟未更新，前台强制弹窗提示“以最终结算页为准”并同步冻结改价权限；单笔尾差≤0.05元自动计入营业外收支平账，严禁前台手工抹零；资金差异率严格控制在0.3%以内。
- 来源涉众：财务人员、客户
- 优先级：中

#### 模块：权限配置与审计日志 (AUD)

**REQ-AUD-001**
- 描述：系统配置变更强制留痕与权限管控。
- 输入：权限/路由/价格参数修改操作。
- 输出：审计日志快照、企微告警。
- 前置条件：系统管理员或授权人员执行配置。
- 验收标准：变更自动留存完整快照（操作人、时间、原值、新值、IP），不可人工关闭；日志支持按门店/字段筛选，检索响应≤3秒；改价超5%、批量删除或权限下发需二次确认并实时企微告警。
- 来源涉众：系统管理员
- 优先级：高

**REQ-AUD-002**
- 描述：高危越权操作前端强阻断。
- 输入：非授权角色触发的跨店强制调单/未授权改价/异常批量导出。
- 输出：阻断弹窗、拦截记录。
- 前置条件：操作员执行超出权限矩阵的动作。
- 验收标准：弹窗显示违规类型、原值快照、一键撤回；误报率控制在1%以内；管理端同步企微告警；常见故障15分钟内凭操作日志精准定位。
- 来源涉众：系统管理员、门店店长
- 优先级：高

### 3.2 非功能需求

| 类别 | 需求描述 | 验收标准 |
|---|---|---|
| 性能 | 文件上传与预览响应 | 上传拦截响应＜2秒，预览加载＜3秒 |
| 性能 | 状态与价格同步延迟 | 页面状态与价格更新延迟＜30秒 |
| 性能 | 日志检索与故障定位 | 配置日志检索响应≤3秒；异常定责与定位≤15分钟 |
| 可用性 | 系统高峰可用性 | 业务高峰期系统可用性不低于99.9% |
| 安全性 | 文件存储与销毁 | 原始设计稿与成品照默认保留7天，提前24小时站内信提醒，到期自动彻底销毁 |
| 安全性 | 高敏订单防冒领 | 单笔＞500元或涉及内部合同订单，取件必须校验手机号后四位或短信验证码，核验率100% |
| 可靠性 | 异常闭环时效 | 跨店调度/人工兜底异常需在4小时内闭环 |
| 合规性 | 电子发票准确率 | 自动触达率≥99%，错误率≤1%，台账与申报表100%一致 |

### 3.3 接口需求

| 接口名称 | 交互方向 | 数据格式/协议 | 核心约束 |
|---|---|---|---|
| 第三方支付接口 | 系统→支付网关/回调 | HTTPS/JSON | 必须匹配商户后台流水号，支持微信/支付宝/对公转账；退款自动回滚原渠道 |
| 企业微信待办接口 | 系统→企微工作台 | HTTPS/Webhook | 审批流转、异常告警、超时升级默认推送至对应角色待办列表 |
| 电子发票税务接口 | 系统→税务平台 | HTTPS/PDF | 按实付金额生成含分项明细电子发票；支持多笔订单合并开票（单次上限50单） |
| 机台扫码拦截接口 | 工位设备→系统 | TCP/MQTT | 接收生产状态、SLA倒计时、冻结指令；支持扫码触发拦截与进度上报 |

### 3.4 数据需求

1. 全生命周期强制落盘字段覆盖：订单基础信息（客户ID、文件规格/页数/纸张克重）、流程节点时间戳（接单至交付各步骤精确到秒）、人员设备绑定（操作员工号、指定机台号）、财务明细（实际折扣额、审批人、支付流水号）、耗材核销（标准用量vs实际领料差异）。
2. 关键字段禁止事后补录或留空；上一节点未闭环时直接阻断下一节点。
3. 周五绩效报表必须包含核心考核指标：五步闭环完成率（目标≥98%）、折扣合规率（目标100%线上留痕）、线上收款占比（目标100%）、交付准时率与客诉率。
4. 配送绩效看板按月自动统计：实际配送单量、准时签收率（目标≥98%）、异常备案数、跨店协作次数。
