# 软件需求规格说明书

**项目名称**：图文快印门店连锁管理系统
**文档版本**：v1.5-draft
**生成日期**：2026-06-18

## 1. 引言

### 1.1 目的
本文档旨在定义图文快印门店连锁管理系统的软件需求规格，明确系统功能、性能、接口及数据约束。本文档作为系统设计、编码实现、测试验证与项目验收的唯一基准，确保所有开发活动可追溯、可量化、可验证。

### 1.2 范围
本系统覆盖图文快印连锁门店的核心业务链路，包含订单接单、文件合规校验、阶梯价自动计算、折扣审批、排产调度、信用额度控制、跨店路由与配送执行、电子发票开具、日结对账、权限管控与全链路审计。系统服务对象包含客户、门店店员、门店店长、总部运营管理员、财务人员、配送外协人员及系统管理员。

## 2. 总体描述

### 2.1 产品视角
本系统为连锁型SaaS平台，包含微信小程序/扫码客户端、门店PC操作端与工位看板、总部运营管理后台、财务对账端、配送外协人员移动端及系统管理员配置端。系统需与第三方支付渠道（微信/支付宝/对公）、企业微信消息推送服务、电子发票税务接口及门店机台扫码设备对接。

### 2.2 产品功能概述
系统核心能力划分为六大模块：订单接单与文件校验、报价与折扣审批、排产调度与信用开工、跨店派单与配送执行、发票开具与日结对账、权限配置与审计日志。各模块严格遵循状态机流转，支持可配置的业务分支（如发票触发节点、路由权重），并强制记录全量操作轨迹。UML活动图已同步修正，确保外协占比拦截与财务核销状态流转分支与SRS文本严格对齐。

### 2.3 用户特征
| 角色 | 核心特征与操作频次 |
|---|---|
| 客户 | 学生/行政/商户，周均下单3至5次，关注报价锁定、进度可见、文件安全与发票自动触达。 |
| 门店店员 | 日均处理40至80单，需3分钟内完成参数核对与报价录入，依赖系统自动防错与状态同步。 |
| 门店店长 | 负责排产调度、折扣特批、异常接管，要求保留应急开关与手动优先级调整权限，每日10分钟内掌握积压与逾期数据。 |
| 总部运营管理员 | 管理30余家门店，制定价格/活动/路由规则，审批超限折扣，监控五步闭环完成率与线上收款占比。 |
| 财务人员 | 处理日结对账与发票开具，要求账实一致、差异精准定位至订单与操作人，日结对账耗时压至30分钟内。 |
| 配送外协人员 | 日均接8至12单跨店调货，依赖首屏关键信息展示与异常在线报备，要求审批超时20分钟自动放行。 |
| 系统管理员 | 管控账号权限、路由权重与基础配置，要求变更100%留痕、高危操作强阻断、故障15分钟内定位。 |

### 2.4 约束条件
1. 强制五步闭环流转，严禁跳步（除应急开关触发外）。
2. 100%线上支付归集，线下收款视为违规。
3. 所有人工干预、改价、应急放行必须强制记录操作人、精确到秒时间戳及业务原因。
4. 系统可用性在业务高峰期不低于99.9%。
5. 所有未决业务冲突已按 CCB 裁决备忘录 v1.0 执行，系统逻辑不再保留模糊分支。

### 2.5 假设与依赖
1. 假设门店网络带宽稳定，支持大文件上传与实时状态同步。
2. 依赖第三方支付平台提供实时交易流水查询接口。
3. 依赖企业微信API实现待办推送与审批流路由。
4. 依赖门店机台支持二维码扫码拦截与状态上报。

## 3. 具体需求

### 3.1 功能需求

#### 模块：订单接单与文件校验 (ORD)

**REQ-ORD-001**
- 描述：客户通过微信小程序或到店扫码提交订单与设计文件，系统执行格式与容量校验。
- 输入：订单基础参数、设计文件（PDF/AI/CDR等）。
- 输出：临时单号、校验通过/拦截结果、缩略预览。
- 前置条件：客户完成身份登录或扫码绑定。
- 验收标准：单文件上限100MB、页数上限80页；超限直接标红拦截并提示“拆分或转PDF”；上传拦截响应＜2秒，预览加载＜3秒；文件全程加密存储。
- 来源涉众：客户、门店店员
- 优先级：高

**REQ-ORD-002**
- 描述：订单满足合规、支付与质检条件后，自动流转至生产状态。
- 输入：支付成功状态或信用额度校验通过标识、前台质检结果。
- 输出：订单状态变更（待审核/待排产/生产中）。
- 前置条件：文件可正常打开且页数/幅面在常规设备范围内；系统报价与到店实价一致；客户已全额支付或信用额度校验通过；门店前台完成初步质检。
- 验收标准：普通订单提交后30分钟内流转，急件≤1小时内流转至“生产中”；条件不满足时订单卡在“待审核/待付款”环节；若触发标准退款流程，资金需在3个工作日内原路返回；系统支持配置VIP特批退款通道，该通道时效不纳入标准SLA基线。
- 来源涉众：客户、门店店员
- 优先级：高

**REQ-ORD-003**
- 描述：订单状态变更时向客户推送进度通知。
- 输入：节点状态事件。
- 输出：客户端消息推送。
- 前置条件：订单进入制作环节。
- 验收标准：客户端界面仅展示“待接单→制作中→已备好（附取件码）”三步状态；系统内部流转状态独立维护；通知延迟≤2分钟；取件前至少提前15分钟明确提醒；支持常用规格一键复购。
- 来源涉众：客户
- 优先级：高

**REQ-ORD-004**
- 描述：文件初审判定为小瑕疵时，执行非阻塞式备注与异步处理。
- 输入：文件解析异常标记、瑕疵类型代码。
- 输出：订单状态保持“待质检”，异步推送瑕疵详情至客户端。
- 前置条件：文件可读取但存在非致命性格式偏差（如字体缺失、局部低分辨率）。
- 验收标准：瑕疵判定耗时≤1秒；订单不挂起、不阻断五步主状态机；系统自动生成非阻塞备注并附加至工单详情页；客户重传合格文件后自动覆盖原瑕疵记录。
- 来源涉众：客户、总部运营管理员
- 优先级：高

#### 模块：报价与折扣审批 (QUO)

**REQ-QUO-001**
- 描述：系统根据录入参数自动计算阶梯报价。
- 输入：纸张克重/材质、黑白/彩色、总页数、工艺（覆膜/装订）、数量。
- 输出：标准报价明细。
- 前置条件：订单参数完整录入。
- 验收标准：常规订单3分钟内自动带出报价；线上明细与到店实付分毫不差；价格推送延迟＜30秒。
- 来源涉众：门店店员、客户
- 优先级：高

**REQ-QUO-002**
- 描述：折扣申请触发分级审批路由。
- 输入：折扣比例或减免金额。
- 输出：审批通过/驳回/升级结果。
- 前置条件：客户申请折扣或店员发起加急附加费调整。
- 验收标准：单笔减免≤15元或实收≥9折，店长直批（响应≤30秒）；超限或实收＜9折，流转总部线上审批；审批超时2小时自动升级至区域经理，再超时2小时直接升级至总部运营负责人并抄送财务备案。严禁店员自助执行折扣逻辑。
- 来源涉众：门店店长、总部运营管理员、财务人员
- 优先级：高

**REQ-QUO-004**
- 描述：折扣区间与金额区间决策矩阵校验。
- 输入：订单实收金额、申请折扣比例、订单总金额。
- 输出：审批路由判定结果。
- 前置条件：触发折扣审批流程。
- 验收标准：系统内置互斥决策表，按优先级顺序执行路由判定：①优先级最高：若单笔减免金额≤15元 或 实收金额≥原价90%，路由至店长直批；②次优先级：若单笔减免金额＞15元 且 实收金额＜原价90% 且 订单金额在800至2000元区间 且 申请折扣减免比例≤5%（即申报折扣档位≤5%），路由至店长直批；③其余所有组合：路由至总部审批。决策表执行耗时≤1秒，结果写入审计日志，各分支在数学与业务上互斥且均可独立触发。
- 来源涉众：总部运营管理员、财务人员
- 优先级：高

**REQ-QUO-003**
- 描述：制作过程中客户临时增加附加服务或触发加价。
- 输入：附加项勾选、价格变动值。
- 输出：更新后总价与剩余预计时间。
- 前置条件：订单处于“制作中”状态。
- 验收标准：总价与剩余时间实时刷新；单笔加价＞5元需二次确认弹窗；未勾选“同意”前不扣款、不进下一环节。
- 来源涉众：客户
- 优先级：中

#### 模块：排产调度与生产流转 (PRO)

**REQ-PRO-001**
- 描述：系统根据订单类型与额度控制排产指令下发。
- 输入：订单类型（全款/信用）、授信额度、财务到账状态、生产进度百分比。
- 输出：机台排产指令或拦截提示。
- 前置条件：订单通过价格校验。
- 验收标准：信用单单笔≤3000元且月累计≤1万元，额度触达80%系统提示风险，100%强制拦截转主管审批；全款单需财务确认到账后解锁；生产进度超60%时触发大单保护锁，禁止强制拦截或改派；触发应急开关强制下发需记录应急原因与审计水印。
- 来源涉众：门店店长、财务人员
- 优先级：高

**REQ-PRO-002**
- 描述：改价或工艺变更审批期间冻结生产作业。
- 输入：审批中状态标识。
- 输出：机台扫码拦截指令、SLA暂停标记。
- 前置条件：店长或总部发起折扣/工艺变更审批。
- 验收标准：车间必须暂停该单扫码作业，严禁转交空闲同事；SLA倒计时同步暂停并标记“待审批冻结期”；解冻后由值班组长手动调回原机台优先插队；审批响应压至30分钟内，冻结期不计入逾期考核。
- 来源涉众：门店店长、总部运营管理员
- 优先级：高

**REQ-PRO-003**
- 描述：突发停工或断料触发人工兜底干预。
- 输入：异常原因码、改派/拆单方案。
- 输出：干预执行结果与对比记录。
- 前置条件：设备故障或核心耗材短缺。
- 验收标准：后台人工兜底最多允许2步（强制改派至指定门店、拆单重审或手工核销）；严禁第3步随意修改单价；强制记录操作人、精确到秒时间戳及业务原因；异常月均发生15至20起需在4小时内闭环。
- 来源涉众：总部运营管理员、系统管理员
- 优先级：高

#### 模块：跨店派单与配送执行 (DLV)

**REQ-DLV-001**
- 描述：跨店调货路由引擎执行派单决策。
- 输入：SLA承诺时效、门店库存、设备负荷、业务资质、财务核销状态。
- 输出：目标派单门店、交付拦截结果、财务状态同步结果。
- 前置条件：触发跨店调度或集中加工。
- 验收标准：以SLA承诺时效为第一阈值；若两家均能按期交付，优先派给库存充足店；资质或设备离线硬性拦截重路由；设备负荷≤60%作为动态均衡次优指标；跨店履约准时率≥95%；当日累计例外金额≤200元时触发批次放行阈值；路由决策执行后，系统需同步将目标订单的 `FinancialVerifyStatus` 枚举值更新为对应状态（覆盖待核销、待核销-耗材、已核销、挂起），确保财务核销状态与调度结果一致；路由引擎权重支持后台配置且变更强制留痕。
- 来源涉众：总部运营管理员、系统管理员
- 优先级：高

**REQ-DLV-002**
- 描述：配送任务下发与途中异常在线处理。
- 输入：任务清单、异常照片/改址申请。
- 输出：签收状态、审批流转结果。
- 前置条件：配送外协人员接单。
- 验收标准：首屏固定显示取送地址与对接人电话、最晚送达/完工截止时间、物品明细与外协工艺要求；派单后3分钟内清单可查；改址/加急审批超时20分钟未批复系统自动放行并标黄预警；签收必扫条码或电子签名。
- 来源涉众：配送外协人员、门店店长
- 优先级：高

#### 模块：发票开具与日结对账 (FIN)

**REQ-FIN-001**
- 描述：电子发票生成触发与逆向红冲控制。
- 输入：取件确认状态/支付流水、退款申请、外协成本占比。
- 输出：电子发票PDF、红冲凭证、拦截提示。
- 前置条件：业务规则配置选定触发节点（交付后开票或付款预开）。
- 验收标准：配置为交付后开票时，扫码取件确认后1小时内推送至指定邮箱；配置为付款预开时，退款触发原发票冻结，红冲必须绑定退款流水号并经财务审批；自动开票上限单笔≤500元且非企业月结账户；跨店分摊或单笔＞3000元拦截转人工复核；外协成本占比（OutsourcingCap）＞15%时系统拦截开票并提示“外协比例超标，需店长审批”；跨店/外协补差账单超时24小时未确认，系统默认按原单金额挂起并推送财务复核。
- 来源涉众：财务人员、客户
- 优先级：高

**REQ-FIN-002**
- 描述：每日闭店自动触发对账并生成日报。
- 输入：当日订单流水、第三方支付渠道账单、仓储计费参数。
- 输出：日结对账报表。
- 前置条件：营业时间结束。
- 验收标准：报表含订单号、操作人、支付方式及渠道单号、应收/实收/手续费、发票状态、退款标记；差异≤100元自动计入营业外收支平账；差异＞100元标红推送店长复核；日结对账耗时压至30分钟内；仓储计费（StorageFeeCap）执行单日≤5元阶梯封顶，超限部分自动计入异常台账。
- 来源涉众：财务人员、门店店员
- 优先级：高

**REQ-FIN-003**
- 描述：计价延迟前端交互与尾差自动处理。
- 输入：计价明细、实付金额、网络延迟时长。
- 输出：弹窗提示、平账流水、前端降级视图。
- 前置条件：客户进入结算页面。
- 验收标准：价格推送延迟＞30秒时前端显示骨架屏与“加载中”状态；延迟＞2分钟时强制弹窗提示“当前网络波动，请以最终结算页为准”，不冻结支付通道，后台同步记录改价日志与时间戳；单笔尾差≤0.05元自动计入营业外收支平账，严禁前台手工抹零；资金差异率严格控制在0.3%以内。
- 来源涉众：财务人员、客户
- 优先级：中

#### 模块：权限配置与审计日志 (AUD)

**REQ-AUD-001**
- 描述：系统配置变更强制留痕与权限管控。
- 输入：权限/路由/价格参数修改操作。
- 输出：审计日志快照、企微告警。
- 前置条件：系统管理员或授权人员执行配置。
- 验收标准：变更自动留存完整快照（操作人、时间、原值、新值、IP），不可人工关闭；日志支持按门店/字段筛选，检索响应≤3秒；改价超5%、批量删除或权限下发需二次确认并实时企微告警。
- 来源涉众：系统管理员
- 优先级：高

**REQ-AUD-002**
- 描述：高危越权操作前端强阻断。
- 输入：非授权角色触发的跨店强制调单/未授权改价/异常批量导出。
- 输出：阻断弹窗、拦截记录。
- 前置条件：操作员执行超出权限矩阵的动作。
- 验收标准：弹窗显示违规类型、原值快照、一键撤回；误报率控制在1%以内；管理端同步企微告警；常见故障15分钟内凭操作日志精准定位。
- 来源涉众：系统管理员、门店店长
- 优先级：高
