# 子技能层资产清单

## 资产清单

| 子技能 | 路径 | 复用场景 |
|---|---|---|
| 需求质量分析 | `wiki/summaries/子技能/需求质量分析/SKILL.md` | A2 前后对 raw notes 或需求清单做质量检查 |
| OpenAPI契约合规校验 | `wiki/summaries/子技能/OpenAPI契约合规校验/SKILL.md` | 生成或修改 OpenAPI 后做结构校验 |
| 设计漂移检测 | `wiki/summaries/子技能/设计漂移检测/SKILL.md` | v1/v2 代码生成后输出 RCR 风格报告 |
| CIA变更影响分析 | `wiki/summaries/子技能/CIA变更影响分析/SKILL.md` | 收到 CR 后执行九维度影响分析 |
| 系统测试报告生成 | `wiki/summaries/子技能/系统测试报告生成/SKILL.md` | 汇总自动化测试、E2E、CRR、compile.js 结果 |

## 复用价值

- 将高频工程操作封装成单一、参数化、结构化输出的流程。
- 便于在 Reasonix、n8n 或通用 LLM 会话中直接调用。
- 每个技能都有验证步骤，减少产物格式漂移。

## 调用方式

1. 在新会话中复制目标 `SKILL.md` 全文。
2. 按“输入参数”表提供项目名、输入文件、输出路径和判定规则。
3. 要求模型严格按“输出格式”生成结果。
4. 使用“验证步骤”逐项自检。
5. 将通过自检的输出写入 Obsidian 对应目录。

## 迁移指南

- 保留技能结构和输出格式。
- 替换输入参数中的项目名称、路径、模块清单和严重程度策略。
- 如果接入 n8n，可将每个技能映射为一个 HTTP 节点或 Code 节点后的 LLM 调用节点。
- 如果接入本地脚本，可把“验证步骤”转为脚本断言。

## 注意事项

- 这些 `SKILL.md` 是课程资产，不是 Codex 本机运行时技能。
- 不应把 API key、GitHub token 或本机绝对私密路径写入技能文件。
- 一个技能只做一件事；需要组合流程时由 n8n 或人工编排。
