# AI 代码生成约束提示词

## 1. 项目记忆
你正在为「图文快印门店连锁管理系统」生成代码。本项目采用**分层架构（模块化单体 Modular Monolith）**，在单一进程和单一数据库实例内运行。系统按业务领域严格划分为 6 个核心模块：ORD(订单接单)、QUO(报价审批)、PRO(排产调度)、DLV(跨店配送)、FIN(财务发票)、AUD(权限审计)，以及 INFRA(基础设施)。所有代码必须遵循高内聚、低耦合原则，严禁破坏模块化边界。

## 2. 契约记忆
你必须严格遵守 OpenAPI 3.0.3 接口契约：
- **DTO 命名与结构**：严格使用契约中定义的 `Order`, `Quotation`, `ProductionTask`, `DeliveryTask`, `Invoice`, `AuditLog` 作为数据传输对象，字段名与类型必须与 YAML 契约中的 `components.schemas` 完全一致。
- **接口路径与 OperationId**：Controller 层的 `@RequestMapping` 路径与 `@Operation(operationId)` 必须与 OpenAPI `paths` 中定义的路径和 `operationId` 精确匹配。
- **错误码规范**：必须使用契约中定义的 HTTP 状态码（如 400, 403, 409, 422）进行异常抛出，禁止自定义非标准 HTTP 状态码。

## 3. 规则记忆
你必须将 TLCD 三层约束文档转化为代码级强制规则：
- **职责约束**：ORD 模块严禁包含价格计算逻辑；QUO 模块的折扣矩阵必须通过配置中心或规则引擎读取，严禁硬编码 `if-else`；FIN 模块的支付与发票操作必须在同一个 `@Transactional` 本地事务中，严禁引入分布式事务注解。
- **接口约束**：模块间通信必须通过 Application Service 接口（如 `OrderAppService`），严禁跨模块注入 Repository 或 DAO。状态流转必须调用 `StateMachine.transit()`，严禁直接 `entity.setStatus()`。外部 API（支付/税务/MQTT）必须通过 INFRA 模块的防腐层调用。
- **依赖约束**：严格遵循 ORD -> QUO -> PRO -> DLV -> FIN 的单向依赖。AUD 和 INFRA 作为底层支撑，严禁反向依赖业务模块。FIN 对 ORD/DLV 的依赖仅限只读查询。

## 4. 生成要求
在生成 Java/Spring Boot 代码时，请遵循以下模板与规范：
- **包结构**：`com.printshop.{module}.{layer}`（如 `com.printshop.quo.application`）。
- **异常处理**：使用全局异常处理器，业务异常必须携带错误码与审计追踪上下文。
- **审计切面**：所有写操作 Service 方法必须添加 `@Auditable` 自定义注解，由 AUD 模块的 AOP 切面自动记录快照。
- **注释要求**：每个类和方法必须包含 Javadoc，明确标注其所属模块、职责边界及对应的 SRS 需求编号（如 `@see REQ-QUO-004`）。

```java
// 示例：符合约束的 Application Service 模板
package com.printshop.quo.application;

import com.printshop.aud.infrastructure.annotation.Auditable;
import com.printshop.quo.domain.service.DiscountRuleEngine;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

/**
 * 报价审批应用服务
 * 职责：阶梯价计算与折扣决策矩阵校验
 * 依据：REQ-QUO-001, REQ-QUO-004
 */
@Service
public class QuotationAppService {

    private final DiscountRuleEngine ruleEngine;

    public QuotationAppService(DiscountRuleEngine ruleEngine) {
        this.ruleEngine = ruleEngine;
    }

    @Auditable(action = "CALCULATE_QUOTATION")
    @Transactional
    public Quotation calculateAndApprove(QuotationRequest request) {
        // 1. 调用规则引擎计算阶梯价，严禁硬编码
        // 2. 执行互斥决策矩阵校验
        // 3. 返回报价结果
        return ruleEngine.execute(request);
    }
}
```
