## 分层结构

- `infra`：审计、统计、状态机、支付/税务/机台防腐层占位适配器。
- `repository`：各业务模块内存仓储。
- `application`：应用服务，承载 v1 业务规则。
- `controller`：REST API 入口，路径与 OpenAPI 契约一致。

## 已实现接口

| 模块 | 接口 |
|---|---|
| ORD | `POST /api/v1/orders` |
| QUO | `POST /api/v1/quotations/calculate` |
| PRO | `POST /api/v1/productions/dispatch` |
| DLV | `POST /api/v1/deliveries/route` |
| FIN | `POST /api/v1/invoices/issue` |
| AUD | `GET /api/v1/audit-logs` |
| INFRA | `GET /stats` |

## 前端演示台

v1 工程新增由 Spring Boot 静态资源目录托管的前端页面：

```text
generated-code/printshop-v1/src/main/resources/static/
```

访问入口：

```text
http://127.0.0.1:8080/
```

页面能力：

- 单独调用 ORD、QUO、PRO、DLV、FIN、AUD 六个业务接口。
- 一键执行 `ORD → QUO → PRO → DLV → FIN → AUD → stats` 完整演示链路。
- 展示最近响应 JSON、审计日志、模块调用次数和 `/stats` 摘要。

## v1 已知简化项

- Repository 使用内存存储，重启后数据清空。
- `/stats` 使用内存统计，重启后清零。
- 支付、税务、企微和机台扫码接口暂用 INFRA 防腐层占位适配器。
- v1 不实现真实数据库事务边界之外的分布式事务、消息队列或外部系统联调。
