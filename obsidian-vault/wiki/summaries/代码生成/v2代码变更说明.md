## 输入来源

## 生成目录

`generated-code/printshop-v2/`

v2 从 v1 增量复制，保留成熟 Print MIS 主流程，同时新增订单变更冻结能力。
在 v2.2 中进一步增加“任务工作台 + 订单聚合详情 + 统一工作流动作入口”，让主流程围绕待办任务推进，减少跨模块跳转。

## 新增代码范围

| 层级 | 文件/模块 | 说明 |
|---|---|---|
| Domain | `OrderChangeRequest` | 记录订单变更前后字段、金额差异、审批状态和冻结时间。 |
| Repository | `OrderChangeRequestRepository` | 查询订单变更请求和待审批请求。 |
| Service | `OrderChangeRequestService` | 创建、查询、批准、驳回订单变更请求。 |
| Service | `OrderChangeGuard` | 在生产和配送动作前阻断待审批变更。 |
| Controller | `OrderChangeRequestController` | 暴露订单变更请求 API。 |
| Migration | `V5__order_change_requests.sql` | 新增订单变更请求表。 |
| Frontend | `orderChangeRequests` 模块 | 支持申请变更和店长审批。 |
| Controller/Service | `WorkbenchController` / `WorkbenchService` | 按角色实时派生待办任务、指标和最近订单。 |
| Controller/Service | `OrderAggregateService` | 汇总订单、报价、作业单、生产、配送、财务、变更、文件和审计时间线。 |
| Service | `OrderWorkflowService.executeAction` | 统一承接提交审核、报价、作业单、排产、完工、配送、签收、收款、开票、退款和申请变更。 |
| Test | `shouldFreezeOrderWhenProcessedOrderChangeIsRequested` | 覆盖冻结、审批和恢复流程。 |
| Test | `shouldUseUnifiedWorkflowWorkbenchAndAggregateOrderView` | 覆盖任务工作台、统一动作入口和聚合详情权限。 |

## 运行方式

```powershell
cd generated-code\printshop-v2
docker compose up -d mysql
mvn spring-boot:run
```

无 Docker 时可使用演示 profile：

```powershell
mvn spring-boot:run -Dspring-boot.run.profiles=demo
```

## 回归验证

```powershell
cd generated-code\printshop-v2
mvn test
mvn package
Get-ChildItem src\main\resources\static\js -Filter *.js | ForEach-Object { node --check $_.FullName }
node scripts\verify-change-regression.js --write
node scripts\verify-change-regression.js --check
```

## 已知简化

- 店长审批为单级审批，暂不接企微或短信通知。
- 变更请求仅覆盖订单关键规格字段，不覆盖配送异常或库存替代料。
- 冻结状态通过订单当前步骤和动作校验体现，未实现独立 SLA 计时器。
